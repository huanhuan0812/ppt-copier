# PPT文件备份监控程序

## 简介

这是一个专为Windows平台设计的PowerPoint文件自动备份工具。当您插入U盘或其他移动设备并编辑其中的PPT文件时，程序会自动将文件备份到本地指定目录，防止数据丢失（老师不保留课件）。

---

## 主要功能

- **自动检测移动设备**：插入U盘或移动硬盘时自动识别
- **实时监控PPT编辑**：通过PowerPoint COM事件捕获文件打开、保存操作
- **智能备份**：只备份有效的PPT文件，自动跳过临时文件（如`~$`开头的文件）
- **按日期归档**：备份文件按日期分文件夹存储，同名文件自动添加序号
- **托盘图标运行**：后台静默运行，右键图标可查看状态和设置
- **可配置选项**：支持自定义备份目录、保留天数、最小文件大小等
- **可选轮询模式**：当COM事件监控失效时，可开启进程扫描作为备用方案

## 配置文件说明

程序首次运行会自动生成 `ppt_monitor.ini` 配置文件，内容如下：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `backup_dir` | 备份文件保存目录 | `./`（当前目录） |
| `max_retention_days` | 备份文件保留天数 | 30天 |
| `enable_fallback_monitor` | 是否启用轮询监控（默认关闭） | false |
| `min_file_size_kb` | 最小备份文件大小（KB） | 10KB |
| `scan_interval_seconds` | 轮询扫描间隔（秒） | 30秒 |
| `auto_start` | 是否开机自启 | false |

1. 启动程序后，会在系统托盘显示图标
2. 插入U盘或移动设备，程序会自动识别
3. 用PowerPoint打开设备中的PPT文件并编辑保存，程序会自动备份
4. 右键托盘图标可查看：
   - 当前连接的移动设备
   - 今日已备份文件数量
   - 轮询监控状态
   - 打开备份文件夹、日志文件夹
   - 切换轮询监控开关

## 实现思路

### 整体架构

程序采用**事件驱动为主、轮询为辅**的双重监控机制，核心分为四个模块：

```
┌─────────────────────────────────────────────────────┐
│                    系统托盘应用                       │
├─────────────────────────────────────────────────────┤
│                   PPTMonitor（核心）                  │
├──────────────┬──────────────┬───────────────────────┤
│ 设备监听模块  │ COM事件模块   │ 进程扫描模块（可选）    │
│ (Windows API) │ (PowerPoint)  │ (psutil进程缓存)       │
└──────────────┴──────────────┴───────────────────────┘
```

### 1. 移动设备检测模块

- 使用Windows API的`RegisterDeviceNotification`注册设备变更通知
- 监听`WM_DEVICECHANGE`消息，识别`DBT_DEVICEARRIVAL`（设备插入）和`DBT_DEVICEREMOVECOMPLETE`（设备移除）
- 解析`DEV_BROADCAST_VOLUME`结构体获取驱动器盘符
- 通过`GetDriveType`判断是否为可移动设备或光驱

### 2. PowerPoint实时监控模块（COM事件）

- 通过`DispatchWithEvents`连接PowerPoint的COM对象
- 订阅三个核心事件：
  - `OnPresentationOpen`：文件打开时触发备份
  - `OnPresentationSave`：文件保存时触发备份
  - `OnPresentationClose` / `OnQuit`：清理缓存资源
- 程序启动时自动扫描已打开的PPT文件，避免漏备份
- COM事件循环运行在独立线程中，不阻塞主界面

### 3. 进程扫描模块（后备方案）

- 默认**关闭**，仅在COM事件监控失效时可手动开启
- 通过`psutil`遍历系统进程，查找`powerpnt.exe`进程
- 调用`proc.open_files()`获取每个PowerPoint进程打开的文件路径
- 筛选出位于移动设备上的PPT文件进行备份
- 采用**进程ID缓存**和**文件列表缓存**机制，减少重复扫描开销（TTL=2秒）

### 4. 性能优化设计

| 优化点 | 实现方式 |
|--------|----------|
| 进程扫描缓存 | 缓存PowerPoint进程ID列表和每个进程打开的文件，2秒内不重复扫描 |
| 状态延迟保存 | 已备份文件记录每30秒保存一次，避免频繁写磁盘 |
| 去重处理 | 通过文件路径+修改时间判断文件是否已备份，避免重复复制 |
| 异步处理 | 所有耗时操作（备份、日志写入、缓存清理）均在独立线程执行 |
| 智能日志过滤 | 默认只记录移动设备上的操作事件，减少日志噪音 |

### 5. 文件备份逻辑

1. **有效性检查**：
   - 文件扩展名是否为`.ppt`/`.pptx`/`.pps`/`.ppsx`
   - 文件名不以`~$`开头（排除临时文件）
   - 文件大小超过最小阈值（默认10KB）
   - 文件未被其他程序独占锁定

2. **去重判断**：
   - 记录已备份文件的路径和修改时间
   - 同一文件在同一天内多次保存，只备份发生变化的版本

3. **备份存储**：
   - 按日期创建子目录（如`2026-04-03/`）
   - 文件名冲突时自动添加序号（`报告_1.pptx`、`报告_2.pptx`）

4. **过期清理**：
   - 每日零点检查并删除超过保留天数的备份文件夹

### 6. 单实例控制

- 使用Windows全局互斥体（`CreateMutex`）确保只有一个程序实例运行
- 再次启动时会激活已有窗口，而非重复运行

## 日志系统

- 日志文件保存在`./logs/ppt_monitor_YYYYMMDD.log`
- 自动轮转：单文件最大10MB，保留最近5个文件
- 日志级别：
  - `INFO`：记录备份操作、设备插拔、事件触发
  - `DEBUG`：记录详细的缓存操作和调试信息
  - `ERROR`：记录异常和错误

## 常见问题

**Q：为什么插入U盘后没有反应？**  
A：请检查U盘是否被系统识别为“可移动磁盘”。某些U盘被识别为本地硬盘时不会触发。

**Q：COM事件监控失效怎么办？**  
A：右键托盘图标 → 设置 → 切换轮询监控，开启后备扫描模式。

**Q：备份文件保存在哪里？**  
A：默认保存在程序所在目录，可通过配置文件`backup_dir`修改。

**Q：程序会占用多少系统资源？**  
A：事件驱动模式下CPU占用极低（<1%）；开启轮询模式后每30秒扫描一次进程，内存占用约20-40MB。
